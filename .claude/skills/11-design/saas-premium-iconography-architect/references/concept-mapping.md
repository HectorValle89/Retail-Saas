# Concept Mapping

Usar esta referencia cuando el concepto sea ambiguo o cuando haya varias opciones validas de simbolo.

## Contenedores

- Usar squircle suave.
- Mantener bordes limpios.
- Usar gradientes lineales suaves.
- Evitar glow pesado o relieves falsos.

## Paletas base

- `Blue-Mist`: azul pastel frio para perfil, calendario, sistema, identidad.
- `Mint-Cream`: verde suave para ventas, progreso, productividad, exito.
- `Soft-Pink`: rosa pastel para amor, lealtad, programas relacionales.
- `Lavender-Haze`: violeta suave para celebracion, obsequios, cumpleaños.
- `Citrus-Cream`: amarillo crema para mensajes, comunicacion, notificaciones suaves.

## Simbolos recomendados por concepto

- Calendario:
  - calendario de hojas
  - reloj secundario si la funcion implica agenda o horario
- Ventas:
  - bolsa de compra
  - carrito solo si el contexto es retail directo
  - grafica secundaria si se quiere enfatizar crecimiento
- Love / Lealtad:
  - corazon limpio
- Comunicacion:
  - sobre
  - burbuja de mensaje secundaria
  - contador solo si el concepto implica notificaciones
- Perfil:
  - persona
  - engrane pequeno solo si se quiere enfatizar gestion personal
- Incidencias:
  - triangulo de alerta
  - pequeno detalle secundario solo si ayuda a comunicar problema operativo
- Incapacidad:
  - corazon con cardiograma
  - cruz roja o simbolo medico si el contexto lo pide
- Vacaciones:
  - palmera
  - silla o sol como apoyo
- Cumpleaños:
  - regalo
  - confeti muy sutil solo si el usuario lo permite

## Criterio de eleccion

- Si el concepto es operativo, priorizar lectura inmediata.
- Si el concepto es emocional, priorizar simbolo limpio y memorabilidad.
- Si el concepto es mixto, usar un simbolo principal y un detalle pequeno.
- Si una composicion tarda mas de un segundo en entenderse, simplificar.
