# Module Priority

## Orden recomendado

1. Shell global
   - `globals.css`
   - `tailwind.config.ts`
   - `layout`
   - `sidebar`

2. Primitives
   - `Card`
   - `Button`
   - `Input`
   - `Select`
   - `Badge`

3. Dashboard
   - Hero
   - KPI cards
   - tablas soft
   - alertas y widgets

4. Ventas
   - Header
   - formulario de captura
   - tabla/listado

5. Empleados
   - Header
   - cards de resumen
   - tablas/listados
   - formularios administrativos

6. LOVE ISDIN
   - Header
   - formulario de captura
   - estados y evidencia

7. Solicitudes
   - Header
   - formulario
   - calendario/listado

## Regla de priorizacion

Si el tiempo es limitado, nunca saltes shell + primitives. Es mejor dejar 5 modulos en 70% pero coherentes, que 1 modulo en 100% con el resto antiguo.
